# 私有网盘系统（FastAPI）功能文档（含页面预览）

本文档基于当前代码与页面实现整理，覆盖现有功能、权限与页面访问入口，并提供“文字版页面预览”（用于验收与沟通设计/交互）。

## 1. 系统概览

- **定位**：个人私有网盘（上传/下载/分享/校验/管理）
- **后端**：FastAPI + SQLAlchemy（当前默认 SQLite；可替换为 MySQL/PostgreSQL）
- **认证方式**：JWT（写入 HttpOnly Cookie：`access_token`）
- **数据安全**：
  - 文件落盘为密文（每文件独立密钥）
  - 分享场景使用 ECDH 派生共享密钥，对文件密钥做二次加密（信封加密思路）
  - 下载时支持完整性校验（SHA-256）
- **前端**：Jinja2 模板渲染（原生 HTML/CSS/少量 JS），统一“极致暗黑禁欲风”设计体系

## 1.1 文件命名与存储策略（重要）

- **文件命名（中文/特殊字符）**：上传时会进行“安全保留”处理：保留中文与常见符号，替换 Windows 不允许字符，禁止路径穿越（`/`、`\` 等）。
- **物理存储隔离**：密文落盘目录按用户隔离：`upload/<user_id>/<filename>`，避免不同用户同名文件互相覆盖。
- **兼容历史数据**：若历史文件仍在旧路径 `upload/<filename>`，首次访问（列表/下载/校验）会尝试自动迁移到新目录（失败则回退旧路径读取）。
- **共享文件重名展示**：在“收到的共享”列表中，若共享文件名与“我自己的文件名”冲突，会自动展示为 `原名 (1).ext`、`原名 (2).ext`… 并在副标题保留原始文件名，避免用户困惑与本地下载覆盖。

## 2. 角色与权限

- **普通用户**
  - 登录/注册/退出
  - 上传文件、查看文件列表、下载、删除
  - 分享文件给其他用户
  - 查看“收到的共享”、下载共享文件、删除共享记录、对共享进行完整性验证
  - 查看账户信息、修改密码
- **管理员用户**
  - 拥有普通用户全部权限
  - 进入管理页查看全站用户列表
  - 删除用户（级联删除：共享记录、文件记录、密钥记录，并删除物理文件）

## 4. 文件功能（核心逻辑）

### 4.1 上传

- **入口（前端）**：`POST /upload`（表单文件字段：`file`）
- **行为**：
  - 校验并规范化文件名（保留中文，替换非法字符，禁止路径穿越）
  - 校验文件类型（白名单扩展名）
  - 限制最大上传大小（按配置）
  - 加密后写入 `UPLOAD_FOLDER/<user_id>/`（密文包含 IV 前缀）
  - 同步计算 SHA-256（针对密文）并写入数据库
  - 文件密钥使用“包裹/解包”机制绑定用户与文件名

### 4.2 下载

- **入口（API）**：`GET /api/files/download/{filename}`
- **校验**：
  - filename 安全校验（禁止路径穿越等）
  - 密文 SHA-256 与数据库 hash 比对（完整性检查）
  - 解包得到文件密钥后流式解密输出
  - 读取/迁移：优先从 `upload/<user_id>/` 读取；若文件仍在旧路径 `upload/<filename>` 会尝试迁移

### 4.3 删除

- **入口（前端）**：`POST /delete_file/{filename}`
- **行为**：
  - 删除数据库文件记录
  - 删除与该文件相关的共享记录
  - 删除物理文件（优先 `upload/<user_id>/<filename>`；若存在旧路径也会清理）

### 4.4 分享

- **入口（前端）**：`POST /share_file/{filename}`
- **流程（概念）**：
  - 发送方与接收方通过 ECDH 派生共享密钥
  - 发送方解包得到文件密钥
  - 用共享密钥加密文件密钥后，写入 shared_files 表

## 5. 完整性验证

- **我的文件批量验证**：`GET /batch_verify`
  - 遍历当前用户文件，计算密文 SHA-256 与库中 hash 比对
- **共享文件单个验证（按共享记录 ID，推荐）**：`GET /verify_shared_by_id/{shared_file_id}`
- **共享文件单个验证（按文件名，兼容）**：`GET /verify_shared_file/{filename}`
- **共享文件批量验证**：`GET /batch_verify_shared`

## 6. API（给前后端分离/自动化用）

说明：以下为主要 API 入口，均需要通过 Cookie 或 Bearer Token 认证（登录后自动具备 Cookie）。

### 6.1 认证

- `POST /api/auth/register`：注册
- `POST /api/auth/login`：登录（设置 Cookie）
- `POST /api/auth/logout`：退出（清 Cookie）
- `GET /api/auth/logout`：浏览器直开兼容（307 跳转到 `/logout`）

### 6.2 用户

- `GET /api/users/profile`：获取当前用户信息
- `POST /api/users/change_password`：修改密码
- `GET /api/users/admin/users`：管理员获取所有用户
- `DELETE /api/users/admin/users/{user_id}`：管理员删除用户

### 6.3 文件

- `POST /api/files/upload`：上传（API 版）
- `GET /api/files/`：列出文件
- `GET /api/files/download/{filename}`：下载（流式）
- `DELETE /api/files/{filename}`：删除（API 版）
- `GET /api/files/{filename}`：浏览器直开兼容（307 跳转到 `/api/files/download/{filename}`）

### 6.4 分享

- `POST /api/share/share/{filename}`：分享给指定用户名
- `GET /api/share/`：列出“收到的共享”
- `GET /api/share/share/{filename}`：浏览器直开兼容（Accept: text/html 时 307 跳转到 `/share_file/{filename}`）
